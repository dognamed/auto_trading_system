# 삼성전자 자동거래 시스템

한국투자증권 Open API를 사용한 삼성전자(005930) 자동거래 시스템입니다.  
**모의 거래 환경 전용**이며, REST API만 사용합니다.

## 📋 프로젝트 구조

```
samsung_auto_trader/
├── main.py              # 프로그램 진입점
├── config.py            # 환경변수 및 설정
├── auth.py              # 토큰 인증 및 캐싱
├── api_client.py        # REST API 추상화 레이어
├── market_data.py       # 현재가 조회
├── account.py           # 계좌 정보 조회
├── orders.py            # 주문 생성/조회
├── trader.py            # 거래 로직 (메인 알고리즘)
├── logger.py            # 통합 로깅
├── token_cache.json     # 토큰 캐시 (자동 생성)
├── requirements.txt     # 의존성
└── README.md            # 이 파일
```

## 🎯 거래 로직

1. **현재가 조회** (005930)
2. **계좌 확인** (잔액, 보유주식)
3. **매수 주문** (현재가 - 2,000 KRW)
4. **매도 주문** (현재가 + 2,000 KRW)
5. **실행 확인** (계좌 다시 확인)
6. **반복** (폴링 주기마다)

## 🕐 거래 시간

- **시작**: 09:10 AM
- **종료**: 03:30 PM (15:30)
- **평일만 거래** (월~금)
- 거래 시간 외에는 자동으로 대기

## 🔧 설치 및 실행

### 1. 환경변수 설정

`.env` 파일을 프로젝트 폴더에 생성하고 다음 내용을 입력합니다:

```env
GH_ACCOUNT=계좌번호
GH_APPKEY=앱키
GH_APPSECRET=앱시크릿
```

또는 시스템 환경변수로 설정합니다:

```bash
export GH_ACCOUNT="계좌번호"
export GH_APPKEY="앱키"
export GH_APPSECRET="앱시크릿"
```

### 2. 의존성 설치

```bash
pip install -r requirements.txt
```

### 3. 프로그램 실행

#### 지속 거래 모드 (일반)
```bash
python main.py --mode run
```

#### 테스트 모드 (한 사이클만 실행)
```bash
python main.py --mode test
```

#### 드라이 런 모드
```bash
python main.py --dry-run
```

## 📊 로깅

모든 주요 이벤트가 다음 위치에 로깅됩니다:

- **콘솔**: `INFO` 레벨 이상
- **파일**: `trading_YYYYMMDD.log` (날짜별 로그 파일)

### 로깅 항목

- 🔑 **인증**: 토큰 발급, 캐시 재사용
- 💰 **현재가**: 각 거래 사이클마다 조회 가격
- 💳 **계좌**: 거래 전후 잔액, 보유주식
- 📤 **주문**: 매수/매도 주문 생성
- ✅ **확인**: 주문 체결 여부 확인
- ⚠️ **에러**: API 오류, 타임아웃, 재시도

## ⚙️ 설정 수정

[config.py](config.py)에서 다음 항목을 수정할 수 있습니다:

```python
# 주문 수량
order_quantity = 1

# 매수/매도 가격 오프셋
order_price_offset_buy = 2000    # 현재가 - 2000원
order_price_offset_sell = 2000   # 현재가 + 2000원

# 폴링 주기 (초)
polling_interval = 10

# 거래 시간
trading_start_time = "09:10"
trading_end_time = "15:30"
```

## 🛡️ 안전 설계

- ✅ 환경변수에서만 자격증명 로드 (하드코딩 없음)
- ✅ 하루 동안 토큰 재사용 (불필요한 인증 호출 최소화)
- ✅ 보수적인 폴링 주기 (모의 거래 API 제한 준수)
- ✅ 재시도 로직 (타임아웃 및 레이트 제한 대응)
- ✅ 거래 시간 확인 (시간 외 주문 불가)
- ✅ 모든 주문 체결 확인 (계좌 다시 조회)

## 🔍 API 응답 필드 수정

API 응답 구조는 환경에 따라 다를 수 있습니다.  
필드명이 다른 경우 다음 파일에서 수정하세요:

| 파일 | 수정 항목 | 참고 |
|------|---------|------|
| [market_data.py](market_data.py) | 현재가 필드명 | `stck_prpr` 등 |
| [account.py](account.py) | 잔액, 보유주식 필드명 | `ord_psbl_cash`, `hldg_qty` 등 |
| [orders.py](orders.py) | 주문 응답 필드명 | `ODNO`, 주문 상태 등 |

## 📌 주요 기능

### 1. 토큰 캐싱
- 같은 날짜 내에서 토큰을 재사용하여 불필요한 API 호출 감소
- `token_cache.json`에 저장

### 2. 에러 처리
- 타임아웃 자동 재시도 (최대 3회)
- 레이트 제한 시 대기 후 재시도
- API 오류 상세 로깅

### 3. 모듈식 구조
- 각 모듈이 명확한 책임을 가짐
- 쉬운 테스트 및 수정
- 의존성 최소화
